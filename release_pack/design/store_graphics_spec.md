# Store Graphics Specification

---

## Google Play

### Feature Graphic (Required)
| Property | Value |
|----------|-------|
| Size | 1024 × 500 px |
| Format | JPG or PNG (no alpha) |
| Max file size | 1 MB |
| Purpose | Shown at top of listing page and in promotional slots |

**Content:** Full-bleed world map or globe imagery with app name overlay.
Do NOT put critical info in outer 10% edges (may be cropped in some views).

### Phone Screenshots (Required: 2 minimum, 8 maximum)
| Property | Value |
|----------|-------|
| Sizes accepted | 320–3840 px, 16:9 or 9:16 ratio (portrait recommended) |
| Recommended | 1080 × 1920 px (portrait) |
| Format | JPG or PNG |
| Max file size | 8 MB each |
| Minimum | 2 screenshots |
| Recommended | 8 screenshots |

### Tablet Screenshots (Strongly recommended)
| Property | Value |
|----------|-------|
| Recommended | 1200 × 1920 px or 2560 × 1600 px |
| Purpose | Shown to tablet users; improves conversion |

## Apple App Store

### iPhone Screenshots (Required)
| Device | Size | Required |
|--------|------|---------|
| 6.9" (iPhone 16 Plus) | 1320 × 2868 px | Yes (newest required size) |
| 6.7" (iPhone 14 Pro Max) | 1290 × 2796 px | Yes |
| 5.5" (iPhone 8 Plus) | 1242 × 2208 px | Yes |

> 6.9" is the newest required size as of 2025. If you only submit 6.7", Apple may auto-scale.
> It is STRONGLY recommended to submit all three sizes.

### iPad Screenshots (Required if supporting iPad)
| Device | Size | Required |
|--------|------|---------|
| 13" iPad Pro (M4) | 2064 × 2752 px | Yes (newest) |
| 12.9" iPad Pro | 2048 × 2732 px | Yes |

### Screenshot Rules (Both Stores)
- No device frames required (but recommended for professional look)
- Text overlays allowed and recommended
- First screenshot is the most important (shown in search results)
- No placeholder or "coming soon" content
- Must accurately represent the current app UI
